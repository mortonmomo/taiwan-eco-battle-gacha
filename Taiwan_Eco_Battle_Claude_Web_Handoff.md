# 《臺灣生態戰》前導網頁版 Claude 接手說明

## 1. 專案定位

目前的網頁版是《臺灣生態戰》的前導抽卡體驗版，用途是宣傳、卡包測試、收藏測試、圖鑑展示與物種圖片測試。正式戰鬥版未來會用 Unity 製作，因此本網頁版不需要實作完整戰鬥、棲地部署、飢餓、技能或 AI 對戰。

目前網頁版應維持為：
- 三包抽卡
- 五連抽與翻牌動畫
- 首發 120 種卡池
- 收藏與圖鑑進度
- Wikimedia 圖片背景載入與 fallback 圖示
- localStorage 保存收藏與圖片快取
- 宣傳與測試用途

## 2. 美術方向

目前暫定美術方向為：

**自然科普 × 卡牌收藏 × 精緻生態 UI**

請維持以下特徵：
- 米白紙張感、森林綠、岩石灰、溪流藍、低飽和自然色
- 精緻但不過度華麗
- 不使用科幻、霓虹、賽博、純像素 UI
- UI 應有自然圖鑑感、收藏感、卡牌包裝感
- 網頁版與未來 Unity 版應能共享視覺語言

## 3. 已套用素材

目前 HTML 已內嵌下列美術素材：
- 遊戲 Logo / 標題字樣
- 高山之巔卡包封面
- 淺山魅影卡包封面
- 森之水脈卡包封面
- 抽卡翻牌用卡背

這些素材是目前前導版使用的第一批視覺素材，可日後替換為正式版，但請不要在沒有指示的情況下移除或改成其他風格。

## 4. 目前已完成的功能

目前版本包含：
1. 三個主題卡包：
   - 高山之巔
   - 淺山魅影
   - 森之水脈
2. 抽卡功能：
   - 每次五連抽
   - 消耗抽卡券
   - 翻牌動畫
   - 4 星、5 星稀有卡有不同視覺效果
3. 收藏系統：
   - localStorage 保存
   - 收藏無上限
   - NEW 標籤
   - 匯出收藏 JSON
4. 圖鑑系統：
   - 可依卡包、星等、關鍵字篩選
   - 顯示首發 120 種物種
   - 顯示收藏進度
   - 顯示物種卡資訊
   - 圖片優先使用 Wikimedia API；缺圖時使用分類圖示 fallback
5. 圖鑑頁強化：
   - 圖鑑總覽列
   - 分類統計
   - 點擊卡牌後開啟生物百科詳情頁
   - 顯示物種名稱、學名、分類、科別、棲地、星等、群落成熟度需求、HP、ATK、描述、圖片來源與複核旗標

## 5. 不要做的事

除非使用者明確要求，請不要做：
- 不要加入完整戰鬥系統
- 不要加入 Unity 相關程式碼
- 不要改變正式名稱《臺灣生態戰》
- 不要恢復 Wiki Battle 字樣
- 不要把能源、能量、PP 加回佈署資源
- 不要自行編造正式物種資料
- 不要刪除 120 種首發卡池資料
- 不要移除 Wikimedia 圖片背景載入邏輯
- 不要把目前 UI 改成科幻、像素、卡通或過度手遊風
- 不要重寫整個 HTML，除非使用者要求重構

## 6. 若要繼續改版，建議優先順序

### 優先項目 A：圖鑑頁深化
可繼續強化：
- 圖鑑卡片點擊後的詳情頁
- 分類篩選
- 棲地篩選
- 圖鑑完成度統計
- 缺圖清單
- 需要人工複核清單

### 優先項目 B：抽卡動畫強化
可繼續強化：
- 卡包開封動畫
- 五張卡飛出動畫
- 稀有卡金光／水波／葉片粒子
- NEW 標籤出現動畫
- 抽卡結果分享圖

### 優先項目 C：宣傳首頁
可新增：
- 首頁 hero section
- 三包展示區
- 玩法介紹區
- 圖鑑與正式 Unity 版預告區
- 問卷或回饋連結

## 7. 修改原則

每次修改請遵守：
1. 採最小修改，不要重寫整個檔案。
2. 保留既有功能。
3. 若新增 CSS，請放在現有 style 後方作為 override，避免破壞原功能。
4. 若新增 JS，請清楚命名函式，避免覆蓋既有函式。
5. 完成後請回報：
   - 新增了哪些區塊
   - 修改了哪些函式
   - 是否影響 localStorage
   - 是否影響 Wikimedia API
   - 如何測試

## 8. 測試清單

每次改完 HTML 後至少測試：
- 頁面可正常打開
- 三個卡包按鈕可抽卡
- 抽卡券會扣除
- 抽卡結果可翻牌
- NEW 標籤正常
- 圖鑑可打開
- 圖鑑卡片可點擊查看詳情
- 搜尋與篩選正常
- Wikimedia 圖片抓不到時不會壞版
- localStorage 保存仍正常

## 9. 目前推薦下一步

若使用者要繼續優化，建議下一步先做：
1. 圖鑑頁更完整：加入缺圖清單與複核清單。
2. 抽卡動畫更完整：加入卡包開封與稀有卡特效。
3. 宣傳頁整合：讓此網頁不只是抽卡工具，也能作為公開展示首頁。
